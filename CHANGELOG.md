# 更新日志

本文件记录 `astrbot_plugin_kook_music` 的重要功能变化和问题修复。

## v1.7.1 - 2026-07-23

### 长视频播放

- 新增 `bili_stream_threshold_minutes`，默认 30 分钟。达到阈值的单个视频/分P以及未知时长内容在轮到队首时 fresh 解析 B站音频 CDN 地址并直接交给 FFmpeg，不再等待完整文件下载；设为 `0` 可恢复全量下载。
- 临时签名流地址与 BVID/分P元数据分离，不会在导入、卡片或日志中暴露。排队、连续切歌、清空、退出和循环重播都会丢弃旧签名，下一次实际播放前重新解析；解析/下载期间切歌会取消当前准备任务，不再等待大文件落盘。
- Direct 与 Relay 两种模式均支持安全 HTTPS 输入；FFmpeg 携带 yt-dlp 返回的 `User-Agent` 与 `Referer`，启用有限网络重连和 15 秒读超时。B站 Cookie 只用于 yt-dlp 鉴权，不会传入 FFmpeg 命令行或 CDN 请求头。
- 数小时流若中途异常退出，会丢弃旧签名、fresh 解析一次，并使用 FFmpeg 输入 seek 从已播放位置前约 5 秒续播；Direct 模式同时刷新 RTP，Relay 模式复用中继。第二次失败才继续后续队列，避免无限重试。
- 未知时长歌曲不再套用 600 秒固定上限，改为等待 FFmpeg 自然 EOF；已知数小时内容仍按真实时长加 30 秒保护。Relay 现在会保留歌曲进程退出码，能区分正常结束与网络/转码失败。

### 卡片与多P修复

- B站播放卡的来源、点歌人、队列状态改为独立 context 行，避免 KOOK 将多个元素横向挤压；超过一小时的时长改为 `H:MM:SS`。
- 解析进度消息不再回显用户粘贴的完整视频 URL，避免 KOOK 或其他链接预览机器人被二次触发。
- 针对 `BV1ANQqBTEVU` 回归确认：该链接是总长 `6:17:29` 的 100P 合集，当前版本展开为 100 个 FIFO 队列项；P1 为 4:57，UP主、分P标题、时长和封面均能正确取得。旧运行实例的总标题、未知UP主、无时长和只播放 P1 现象不再沿用。
- FFmpeg 启动失败及后台 stderr 统一脱敏 HTTPS 地址，防止错误文本回显 B站 CDN 签名参数。

### 验证

- 通过 106 项仓库自动化测试，覆盖 100P 展开、长/未知时长流式选择、短视频下载回退、Direct/Relay 参数与日志脱敏、异常断流 fresh 解析与断点续播、未知/3 小时时限、循环刷新签名、并发入队清签名、连续切歌、小时格式、卡片布局和解析提示。
- 联网实测 P1 fresh 解析得到 HTTPS 音频流，仅向 FFmpeg 传递 `Referer`/`User-Agent` 后连续实时读取 3 秒并正常退出；元数据为 UP主“那首你最爱的歌谣啊”、时长 297 秒且封面为 HTTPS。

## v1.7.0 - 2026-07-23

### 新增

- B站命令由 `播放b站` 简化为可扩展的 `播放`，支持关键词、BV号、视频链接、分P链接、收藏夹链接和 `b23.tv` 官方短链。
- 未显式指定 `?p=N` 的多P视频会按原始分P顺序整批加入队列；显式分P只播放对应一P。收藏夹中的多P视频默认只加入 P1，避免递归展开。
- 支持空间收藏夹、`medialist` 与 `list/ml...` 链接；私密收藏夹可复用 `bili_cookie` 中的 SESSDATA 鉴权。
- 大型收藏夹和多P视频复用歌单区间选择、超时取消、实时容量复检与整批原子入队。收藏夹先建立轻量占位，只请求最终选中区间覆盖的分页。

### 修复与安全

- B站元数据提取与音频下载统一启用 `noplaylist`，并在下载时从已验证的 BVID 与页码重建官方 URL，避免 yt-dlp 意外展开整套分P或播放错误分P。
- 修复多P歌曲卡片把内部 `_pN` ID 拼进链接导致跳转无效的问题；播放卡现在始终指向精确的 `?p=N` 页面。
- B站 API 增加全局串行节流、412/429/5xx 与 `-412/-509/-352` 风控码退避重试、收藏夹同页请求合并和短时缓存，降低大型收藏夹触发风控的概率。
- 关键词播放优先解析 B站服务端渲染搜索页，并保留官方搜索 API 与 yt-dlp 兜底，避免 `bilisearch1` 单一路径遇到 HTTP 412 后完全不可用。
- 修复非法 `p=`、`p=abc`、`p=0` 被误当成“未指定分P”并展开全集的问题；官方短链采用限制跳转次数与域名白名单的手动展开。
- Cookies 文件只读取精确 B站域名下的必要 Cookie，拒绝 `evilbilibili.com` 等相似域名；收藏夹失效、删除或非视频项目会单项跳过而不中断整个批次。
- 修复慢请求可能晚到并反向替换较新请求、以及请求在语音频道查询期间被替换后仍然入队的竞态；最终 marker 校验与原子入队现在使用同一提交锁线性化。
- 视频详情的成功与失败短时缓存都限制为最多 256 项，避免随机无效 BV 输入让负缓存持续增长。

### 队列与下载行为

- 收藏夹/多P批次通过 `join_and_play_many` 追加到已有点歌之后，保持 FIFO；容量不足时整批拒绝，不会产生部分入队。
- 批量添加阶段仅保存视频元数据，不会预下载全部音频；每一项轮到队首后才由 yt-dlp 下载，连续切歌仍沿用现有播放任务与跳过计数逻辑。

### 验证

- 自动化测试覆盖 B站链接与伪域名、非法分P、官方短链、多P顺序、显式分P、收藏夹占位与按页物化、同页请求合并、私密收藏夹、失效条目跳过、Cookie 域名边界、`noplaylist` 和精确播放卡链接。
- 通过 83 项仓库自动化测试、`compileall`、配置 JSON 校验和差异检查；新增覆盖请求替换期间禁止旧批次提交、负缓存上限和服务端渲染搜索结果解析。
- 联网冒烟测试确认关键词“海阔天空”解析为 `BV1WKg3z5Ehi`，`BV1dujdzrEA4` 可按顺序展开 5 个分P，显式 P2 只下载对应的 `563,238` 字节 M4A；公开收藏夹 `213003412` 读取到 7 项并仅物化选中的前三项，`https://b23.tv/BV1dujdzrEA4` 可安全展开到官方长链接。

## v1.6.0 - 2026-07-19

### 新增

- QQ 音乐官方匿名 vkey 返回会员拒绝后，可按原始 `songmid` 调用可配置的会员解析源；默认使用 `https://meting.mikus.ink/api`，配置留空即可禁用。
- 兼容 Meting API 与 `api/song/url` 响应格式，便于切换到自建解析服务；后者可配置 `128`、`320` 或 `flac` 请求音质。
- 配置会员解析源后，QQ 搜索会保留付费歌曲候选；未配置时继续只显示匿名可完整播放的歌曲。

### 安全与兼容

- 免费歌曲始终优先走 QQ 官方播放链路，会员解析仅作为官方拒绝或临时失败后的同 ID 兜底。
- Meting 解析先校验显式 `songmid` 或 URL/LRC 中的歌曲 ID，再禁止自动跟随重定向；其他解析响应同样必须精确包含请求 MID，禁止按同名歌曲替换。
- 解析成功后保留 QQ 官方标题、歌手、时长和封面，并清除旧的会员拒绝原因；解析源失败时仍保留官方拒绝结果，临时故障才继续现有同 ID 备用链路。
- 支持把自建解析 API 配置为本机或内网地址，但最终音频必须跳转到 QQ 官方 HTTPS 音频 CDN；同时拒绝十进制整数、缩写和八/十六进制形式的私网 IP 地址。

### 联网验证

- QQ 会员曲 `000WdUWo1p4MAt`（Russ《Psycho,Pt. 2》）经默认解析源取得 QQ CDN 的完整 M4A：`1,988,937` 字节，`ffprobe` 时长 `162.052` 秒；不是官方匿名链返回的 60 秒试听片段。
- 免费对照曲 `001iLFTh0fcAdb` 仍由 QQ 官方匿名 vkey 直接解析，没有经过会员解析源。
- 禁用会员解析源后，会员曲仍按原逻辑返回 `resolver_status=denied`，不会误播试听或同名版本。
- 通过 63 项自动化测试，并通过 `compileall`、配置 JSON 校验和差异检查。

### 已知限制

- 默认公共解析服务依赖服务方的 QQ 音乐会员 Cookie，没有可用性保证；长期部署建议自建 `mikus-loli/Meting-API` 并配置自己的有效会员 Cookie。
- 默认 Meting 源当前为完整 C400 AAC（约 96 kbps），不是无损音质；`qq_vip_resolver_quality` 仅对支持该参数的 `api/song/url` 类解析源生效。

## v1.5.1 - 2026-07-19

## v1.5.0 - 2026-07-19

### 新增

- `导入歌单` 扩展到 QQ 音乐与酷狗音乐。QQ/酷狗官方歌单链接可自动识别；数字 ID 可使用 `导入歌单 qq <ID>`、`导入歌单 酷狗 <ID>`，裸数字继续默认网易云以保持兼容。
- 支持新版酷狗 `gcid_...`、旧版数字 `specialid` 以及 `collection_...` 歌单 ID，包括 `specialid=0` 的用户自建歌单。
- QQ/酷狗歌单沿用大歌单区间选择、超时取消、实时剩余队列容量检查和整批原子入队；最大队列配置仍支持 2000 首。
- 歌单导入结果卡片显示来源平台；三平台歌曲可以与单曲点歌按 FIFO 顺序混合播放。

### 修复与兼容

- QQ 歌单优先使用可返回完整原始顺序的官方接口，并严格校验返回总数；分页兜底按固定页长推进，平台过滤或无效项保留不可播放占位，避免用户选择的歌单序号偏移。
- 酷狗歌单优先读取 `nofilt` 完整列表并校验分页总数、更新时间和原始顺序；签名接口失败时再使用 `song_v2` 过滤列表兜底。
- QQ/酷狗歌曲只在实际轮到播放时刷新临时播放地址，不会在导入 200/1000 首歌单时提前下载全部音频；会员、付费、版权或下架条目会被跳过并继续后续队列。
- 仅网易云歌单执行网易云元数据补全，避免将 QQ `songmid` 或酷狗 `FileHash` 错发给网易云详情接口。
- 修复 QQ 超过 1000 秒的长音频时长被误当成毫秒的问题，并保留旧版无协议 `playlist?id=...` / `djradio?id=...` 输入兼容。
- 歌单链接解析严格校验官方域名，避免相似恶意域名被当作 QQ/酷狗链接。

### 验证

- 通过 51 项自动化测试，新增覆盖三平台歌单链接解析、QQ 完整列表与分页回退、酷狗 Android/Web 固定签名向量、`gcid` 解码、数字 ID 转换、完整列表分页、不可播放占位和平台结果卡片。
- 联网验证 QQ 1345 首歌单、酷狗 100+ 首普通歌单、`specialid=0` 用户歌单和 952 首大型歌单均能保持顺序导入；QQ 免费曲目可懒解析播放，会员曲目会明确拒绝而不中断队列。

## v1.4.0 - 2026-07-19

### 新增

- QQ 音乐搜索、歌曲详情和匿名可播放地址解析。使用 QQ 音乐网页接口获取稳定 songmid，并在播放前按 media_mid 刷新临时 vkey。
- 酷狗音乐搜索和匿名可播放地址解析。使用酷狗网页搜索及播放接口，以 FileHash 保持歌曲身份并在播放前刷新临时地址。
- 支持常见 QQ 音乐单曲链接、酷狗带 Hash 链接以及新版酷狗单曲网页链接直接点歌。
- 搜索结果和正在播放卡片显示中文音乐平台名称。

### 修复

- 修复第三方聚合接口从 `songs` 改为 `data` 后 QQ 音乐搜索始终为空的问题。
- 修复原聚合接口酷狗搜索返回 404，导致配置中虽然存在酷狗但实际无法点歌的问题。
- 修复将 QQ 的会员权益字段以及酷狗的 Privilege/PayType 直接当作不可播放标志，误删实际可播放歌曲的问题；现在以平台播放接口返回的完整 URL 为准。
- 修复平台解析暂时失败时无法进入同歌曲 ID 备用解析源，以及备用接口返回其他 ID 时可能播错歌曲的问题。
- 修复旧式 QQ 单曲链接可能被网易云正则截断并误识别为网易歌曲的问题。
- 修复 QQ 临时 M4A 地址仍按 `.mp3` 后缀保存的问题；下载器现在按响应类型保存 MP3/M4A/AAC/OGG/FLAC/WAV。
- 修复解析站返回空 HTML、图片或伪装二进制内容时仍被保存并交给 FFmpeg 的问题。
- 搜索选歌等待状态按用户、服务器和文字频道隔离；同频道的新搜索会安全替换旧搜索，不再覆盖 Future 或误删新卡片。

### 限制与验证

- QQ/酷狗仅播放平台允许匿名完整播放的资源。会员、付费、版权或地区受限歌曲会给出明确原因，不会把几十秒试听片段伪装为完整歌曲。
- 本版本扩展的是单曲搜索、直接链接和混合队列播放；`导入歌单` 目前仍为网易云歌单/电台功能。
- 通过 37 项自动化测试，覆盖平台字段兼容、稳定 ID 刷新、付费拒绝、错误 ID 防护、M4A 下载、无效响应拒绝和网易云/QQ/酷狗混合播放流。
- 联网冒烟测试实际完成 QQ MP3（4,190,664 字节）与酷狗 MP3（3,047,954 字节）下载，并验证两平台常见单曲链接解析成功。

## v1.3.1 - 2026-07-19

### 新增

- 支持超大网易云歌单按歌曲序号区间导入。选择区间必须小于等于当前队列剩余容量，超时后自动结束本次导入。
- 歌单最大队列配置上限由 200 提升至 2000，并增加歌单区间选择等待时间配置。
- 歌单批量入队改为原子操作：容量足够时整批追加，容量不足时整批拒绝，不会产生只加入部分歌曲的队列。

### 测试中发现并修复

- 修复等待歌单区间时拦截普通聊天或其他命令，以及与搜索结果数字选择冲突的问题。
- 修复同一用户在不同频道导入歌单时可能共享等待状态，以及连续发起导入时旧请求覆盖新请求的问题。
- 修复极长数字区间输入可能触发整数转换异常的问题。
- 修复大歌单导入时过早获取全部歌曲时长造成额外等待的问题；现在仅补全最终选中区间的歌曲信息。
- 修复第三方歌单接口返回不完整时歌曲缺失或顺序不准确的问题；使用网易云官方歌曲 ID 顺序校验并补齐结果。
- 修复大队列卡片可能超过 KOOK 卡片模块数量限制的问题。
- 修复单曲循环模式下点击跳过仍重复当前歌曲的问题。
- 修复短时间连续点击多次跳过可能只生效一次的问题；每次跳过现在对应消费一首歌曲。
- 修复下载失败与跳过同时发生时可能额外丢弃下一首歌曲的问题。
- 修复 Direct 模式在跳过或启动播放失败后没有及时刷新 RTP 连接，导致后续歌曲无法正常推流的问题。
- 修复首次并发点歌可能重复创建语音会话，以及加入语音过程中退出时残留会话的问题。
- 修复清空队列、取消下载或 B 站后台下载被取消时可能残留临时音频文件的问题。
- 修复超长队列中的旧播放链接失效后无法恢复的问题；下载失败时会重新解析歌曲链接并重试一次。
- 修复语音连接连续恢复失败后队列仍保持非空、播放任务卡死的问题。
- 修复运行时队列上限和等待时间配置缺少类型与范围保护的问题。

### 验证
