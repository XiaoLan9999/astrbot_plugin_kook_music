# astrbot_plugin_kook_music

面向 KOOK 平台的 AstrBot 语音点歌插件。插件会自动加入点歌用户所在的语音频道，通过 FFmpeg 推送音频，并提供 KOOK 卡片按钮控制播放。

## 功能

- 网易云、QQ 音乐、酷狗、酷我、咪咕、百度等平台搜索
- 网易云单曲、歌单和电台节目导入
- B 站关键词、视频、分P、收藏夹和官方短链音频播放
- 播放队列、插队、切歌、循环模式和清空队列
- KOOK 卡片按钮控制下一首、循环和清空
- relay/direct 两种语音推流模式
- 严格限定 KOOK 平台；其他平台命令静默忽略

## 下载策略

导入歌单时只获取歌曲元数据并加入播放队列，不会立即批量下载全部歌曲。每首歌轮到队首后才会下载到本地缓存，下载完成后再开始播放。

这可以避免导入大型歌单时立即占用大量带宽和磁盘，但两首歌之间的等待时间会受到下一首歌曲下载速度影响。目前没有后台并行预下载下一首。

当歌单歌曲数超过当前队列的剩余容量时，插件会要求用户输入 `起始-结束` 的 1-based 闭区间，例如 `201-400`。区间长度不能超过实时剩余容量；等待期间如果其他用户继续点歌，插件会按最新队列容量重新校验。选择超时后自动取消本次导入。

## 安装

1. 在 AstrBot 插件市场通过仓库地址安装，或下载仓库 ZIP 后上传安装。
2. 启用并配置 KOOK 平台适配器。
3. 确保 Bot 具有访问文字频道、语音频道和发送消息的权限。
4. 插件会优先使用自定义 FFmpeg 路径，否则尝试使用或安装 FFmpeg。

## 命令

| 命令 | 说明 |
| --- | --- |
| `点歌 <歌名> [平台]` | 搜索并播放歌曲 |
| `播放 <关键词/BV号/视频或收藏夹链接>` | 播放 B 站视频音频，支持多P与收藏夹批量添加 |
| `导入歌单 <ID或链接>` | 导入网易云、QQ 音乐或酷狗歌单 |
| `下一首` | 跳过当前歌曲 |
| `歌单` | 查看播放队列 |
| `队列插队 <序号>` | 将指定歌曲移动为下一首 |
| `循环模式` | 切换关闭、单曲、列表和随机模式 |
| `清空歌单` | 清空等待队列并保留当前歌曲 |
| `退出语音` | 退出语音频道并清空队列 |

### B 站批量添加

- 未指定 `?p=N` 的多P视频会按 P1、P2、P3 的原始顺序整批加入队列；显式指定 `?p=N` 时只添加该分P。
- 支持空间收藏夹、`medialist`、`list/ml...` 以及 `b23.tv` 官方短链。收藏夹内的多P视频默认只添加 P1，避免递归展开导致队列意外膨胀。
- 收藏夹或多P数量超过实时队列剩余容量时，会要求输入 1-based 闭区间，例如 `201-400`；只读取最终选中区间覆盖的收藏夹分页。
- 批量添加只写入元数据，音频仍在轮到播放时才下载；失效或已删除的视频会跳过并显示数量。

示例：

```text
播放 BV1dujdzrEA4
播放 https://www.bilibili.com/video/BV1dujdzrEA4?p=2
播放 https://space.bilibili.com/84912/favlist?fid=213003412
```

## 配置

- `default_platform`：默认搜索平台
- `streaming_mode`：`relay`（推荐）或 `direct`
- `max_queue_size`：单服务器最大队列长度，配置上限 2000
- `playlist_range_timeout`：大型歌单等待用户输入歌曲区间的超时时间
- `max_sessions`：最大并发语音会话数
- `auto_leave_timeout`：队列空闲后的自动退出时间
- `volume`：FFmpeg 输出音量
- `search_limit`：搜索结果数量
- `custom_ffmpeg_path`：自定义 FFmpeg 可执行文件
- `kook_token`：无法从 KOOK 适配器读取 Token 时手动填写
- `bili_cookie`：可选的 B 站 SESSDATA，用于需要登录的视频和私密收藏夹

请勿将 Token、Cookie 或 AstrBot 运行时数据提交到仓库。
